# After-Sales Service Coordination Tracker

This project demonstrates a simplified after-sales support workflow used to manage customer service requests.

The tracker simulates how service teams coordinate customer inquiries, prepare spare-parts offers, and escalate technical issues to engineers.

## Key Components

Customer Request Tracking  
A structured log for incoming customer inquiries including request status, priority, and follow-up actions.

Spare Parts Offer Management  
Tracks supplier prices, additional costs, and final offer prices sent to customers.

Technical Escalation Log  
Records technical questions escalated to engineers and their resolutions.

Service Dashboard  
A simple operational dashboard summarizing:
- Total customer requests
- Open service cases
- Offers sent
- Confirmed orders
- Technical escalations

## Purpose

This project reflects the type of structured coordination and tracking work I supported in previous roles, such as maintaining organized overviews of requests, tracking task progress, and supporting communication between teams.

The dashboard simulates how customer inquiries, spare-parts offers, and technical escalations can be monitored in an after-sales service workflow.

All data used in this project is fictional and created purely for demonstration purposes.
